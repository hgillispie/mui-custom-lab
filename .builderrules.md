| Component Type | Source | Rationale |

## üé® Sage & Pine Design Token Reference

### Colors (CSS Variables)
- `--color-primary-500` - Pine `#31473a` (Dark green primary)
- `--color-secondary-500` - Sage `#7c8363` (Medium green secondary)
- `--color-success-500` - Success states
- `--color-warning-500` - Warning states
- `--color-error-500` - Error states

### Typography (Font System)
- `--font-family-body` - Outfit (body text)
- `--font-family-heading` - Poppins (headings)

### Component Usage
Our components automatically use these tokens, MUI components inherit the font system.

## üí° AI Best Practices

### Do's ‚úÖ
- Use our components for all interactive UI elements
- Use MUI directly for layout (Box, Stack, Grid)
- Use MUI Typography for text (inherits our fonts)
- Follow consistent prop naming patterns
- Implement loading, error, and success states
- Use templates for complete page implementations

### Don'ts ‚ùå
- Don't import MUI components we've transformed
- Don't skip error handling in forms
- Don't ignore responsive design
- Don't hardcode colors instead of using our components
- Don't mix component systems unnecessarily
